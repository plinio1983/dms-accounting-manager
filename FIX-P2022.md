# Fix errore P2022 paymentStatus

L'errore:

```text
The column `paymentStatus of relation Expense` does not exist in the current database.
```

significa che Prisma Client è stato generato sul nuovo schema, ma il database PostgreSQL contiene ancora la vecchia tabella `Expense`.

## Soluzione consigliata in locale

```bash
npm run db:push
npm run db:seed
npm run dev
```

`db:push` sincronizza direttamente il database con `prisma/schema.prisma`, aggiungendo le colonne mancanti.

## Se vuoi ripartire da zero in sviluppo

Questo cancella e ricrea i dati locali:

```bash
npm run db:reset-local
npm run db:push
npm run db:seed
npm run dev
```

## Nota

Per un progetto in produzione useremo migration versionate; in questa beta locale `db:push` è più pratico perché stiamo ancora cambiando spesso il modello dati.
